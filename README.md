# Azure-Cloud-SOC-Lab

## Objective 
Build a cloud-based environment to simulate attack activity and practice threat detetction 

### Environment 
- Azure Virtual Machine (Windows Server target)
- Kali Linux attacker machine (VMware)
- Configured NSGs and firewall rules
- Enabled diagnostic logging to Log Analytics

### What I Performed 
- RDP brute-force simulation
- Log collection and KQL queries
- Monitoring failed login attempts
- Incident documentation workflow

### Key Takeaways 
- Importance of log visibility
- How exposed RDP increases risk
- Cloud misconfigurtion as attack surface
